# Design Workflow

## Role
Define what good looks like before planning and build.

## Current minimal behavior
- list evaluation-stage projects that are ready for design
- build a Design Package from the latest Review Package
- store that design package in the database
- write a matching Obsidian design note
- move the project from `evaluation` into `design`

## Current posture
This first pass keeps Design narrow:
- no planning breakdown yet
- no task generation yet
- no dashboard expansion yet
- just a real design handoff package and stage transition
