# Repo Strategy

## Purpose
This repository exists first as a credibility-facing package for Greg's public professional image.

That means the main goals are:
- look professional
- show real thoughtful work
- be understandable to a visitor quickly
- present modest but impressive engineering judgment

## What matters most
- a strong README
- clear scope boundaries
- visible progress narrative
- clean repo structure
- practical screenshots and proof points later

## What matters less initially
- flashy framework complexity
- volume of code for its own sake
- noisy commit activity without narrative

## Public transparency model
Internally, Project Manager remains the deeper planning/source-of-truth layer.
GitHub can later carry a lightweight public-facing progress board for visibility.
